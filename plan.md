작성일: 2026-07-08 · 상태: 설계 확정 대기

## 1. 개요

https://www.ai-night.study/tutorials 의 Day 1-20 커리큘럼 전면 재설계.

- **타겟**: AI가 처음인 소상공인
- **운영**: 강사 동반 라이브 수업 (어려운 날은 강사 도움 전제)
- **도구**: Claude 데스크톱 앱 / Codex 앱 (설치·설정만 앱별 분기, 이후 통합 서술). 데스크톱 앱 안에서 에이전트 기능(파일 작업·스킬·MCP·예약) 전부 사용
- **하루 분량**: 25-30분, 매일 실물 결과물 1개

1. **매일 실물 결과물 1개** — 수업 끝에 손에 잡히는 것이 남는다

3. **습관은 중요도 순으로 등장** — 중요한 습관일수록 앞 주차에

## 4. 주간 히스토리 추출 설계

### 기본 흐름
매일은 지금 대화에 "오늘 만든 것"과 "다음에 이어 할 말"만 짧게 남긴다.
D5·D10·D15·D20에는 Claude/Codex 앱 대화 히스토리를 먼저 확인하고, 접근이 안 될 때만 `ai-study/outputs`와 현재 대화를 바탕으로 AGENTS.md 후보를 추출한다.

### 폴더 구조

```
ai-study/                  ← 작업 폴더 (Day 1에 생성)
├── AGENTS.md              ← 내 가게 설명서 (Day 4에 생성, 매주 갱신)
└── outputs/               ← 파일 저장이 필요한 결과물의 보조 보관함
```

- 매일 마지막 화면은 지금 대화에 만든 것과 다음에 이어 할 말 1줄만 남긴다.
- 파일 저장은 인보이스, 장부, 웹페이지 링크처럼 실제 파일이 필요한 본문 단계에서만 안내한다.
- 앱 히스토리는 정적 웹사이트가 직접 읽지 않는다. 사용자가 Claude/Codex 앱에 복사해 넣는 프롬프트가 수행한다.
- AGENTS.md는 **짧게, 검증된 것만** 유지한다.

### 매일 닫기 프롬프트

오늘 만든 것을 대화 안에 짧게 남긴다. 파일 저장은 기본 경로가 아니다.

```markdown
- 오늘 만든 것:
- 다음에 이어 할 말:
```

### 주간 히스토리 추출 (D5·D10·D15·D20)

1. 먼저 Claude/Codex의 최근 대화·작업 히스토리를 읽을 수 있는지 확인한다.
2. 읽기 전에 어떤 히스토리 또는 폴더를 볼지 사용자에게 말한다.
3. 접근이 안 되면 `ai-study/outputs`와 현재 대화에서 확인 가능한 내용만 쓴다.
4. 비밀번호, 토큰, 결제정보, 개인정보는 후보에 넣지 않는다.
5. 확실하지 않은 내용은 `확인 필요`로 표시한다.
6. **추가·수정 제안을 보여주고, 사용자가 승인한 것만 반영**한다.

→ 이 루프가 self-improving / personalized AI의 뼈대: 결과물과 실제 대화에서 패턴을 뽑고, 설명서는 매주 승인해서 반영한다.
