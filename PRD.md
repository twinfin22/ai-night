## 0. One-line

비수도권 지방 재단·청년센터의 30대 주무관이 **60초 안에** "이 강사를 결재 라인에 올릴 수 있다"고 판단하게 만드는, **PDF 소개서를 중심에 두는** 1페이지 강사 포트폴리오 사이트.

---

## 1. Why this exists

**기존 강사 사이트의 잘못된 가정** — "사이트가 컨버전 지점"이라고 봄.
**현실** — 주무관의 진짜 작업물은 **결재 문서 붙임**. 사이트는 그 PDF로 가는 자판기일 뿐.

따라서 본 사이트의 1차 KPI는 **PDF 다운로드 수**, 2차 KPI가 문의 수.

---

## 2. Target reader & 60-second journey

### Reader
- 비수도권 광역시·도 산하 재단 / 시·군 청년센터의 30대 주무관
- 의사결정 권한 없음. 상사 (팀장→과장→부장)에게 결재 올림
- "내 기관과 비슷한 규모의 사례"와 "단가 한도 내 산정 가능"이 통과 조건

### 60초 동선 (이 동선을 사이트가 강제해야 함)

```
0:00 ─ 헤더 진입
0:05 ─ 헤드라인 1줄로 "내 기관 대상 강사구나" 인지
0:15 ─ 출강 기관 로고 스트립으로 신뢰 검증
0:25 ─ 대표 강의 주제 3-4개 훑음
0:35 ─ 비슷한 규모 사례 1건 찾음 (인용 가능한 한 줄 포함)
0:45 ─ "공공기관 단가 준수" 문구 확인
0:55 ─ PDF 소개서 다운로드 → 결재 라인으로 갈 준비 완료
```

이 동선 중 어느 한 칸이라도 5초 이상 걸리면 이탈. 디자인은 이 시계를 우선한다.

## 3. Instructor positioning (the 한 줄)

- "AI 엔지니어 출신이 아니라 **눈높이를 맞추는 교육자**"
- 차별화 축: **비전공자/지방 실무자 대상 AI 교육 전문성**
- 안티-포지셔닝: "혁신적인 AI 전문가" / "차세대 AI 강사" / "열정적인 교육자"
- 톤: 공문 호환. 마케팅 영어 금지 (innovative, cutting-edge, next-generation 류 사용 금지)

**Hero headline 후보 (PRD 단계 — copy는 build 단계에서 확정):**
> "지방 공공기관 실무자를 위한 AI 업무자동화 강의 — 누적 N회 출강, 평균 만족도 X.X"

## 4. Information architecture

**페이지 수: 1 (single-page).** 별도 라우트 없음. PDF 다운로드 = 진짜 "2페이지".

```
┌──────────────────────────────────────────┐
│ § Hero                                   │  ← 5초
│   - 한 줄 헤드라인                          │
│   - 신뢰 라인 (누적 출강·기관 유형·만족도)    │
│   - PRIMARY CTA: 강의소개서(PDF) 다운로드    │
│   - SECONDARY CTA: 카카오톡 채널              │
│   - 강의 현장 액션샷 1장 (스튜디오 X)         │
├──────────────────────────────────────────┤
│ § Trust strip (로고 행)                   │  ← 10초
│   - 남해 청년센터 / 청도 경북시민재단 등     │
├──────────────────────────────────────────┤
│ § 누가 이 강의를 듣나                       │  ← 15초
│   - "이런 분께 추천합니다" 3-bullet         │
│   - 지방 재단/청년센터/공공기관 명시         │
├──────────────────────────────────────────┤
│ § 대표 강의 주제 (3-4개 카드)                │  ← 25초
│   - 주제 / 학습 결과 한 줄 / 대상 / 시간       │
├──────────────────────────────────────────┤
│ § 출강 사례 (3-5건)                         │  ← 35초
│   - 기관·규모·일정·시간·인원·만족도·후속     │
│   - 명시된 인용 1줄 (실명·기관·연도)          │
├──────────────────────────────────────────┤
│ § 강사료 정책 (1줄)                         │  ← 45초
│   - "공공기관 강사료 지급기준 준수 ·         │
│      청탁금지법 한도 내 산정"               │
├──────────────────────────────────────────┤
│ § 강사 소개 (3-5줄)                         │  ← 50초
│   - 약력. 학력은 마지막 줄.                  │
│   - 자격 (예정 포함, "예정" 명시)            │
├──────────────────────────────────────────┤
│ § 문의 (footer)                            │  ← 55초
│   - PDF 다운로드 (재노출)                    │
│   - 카카오톡 채널 / 이메일 / 휴대폰          │
└──────────────────────────────────────────┘
```

**제외 (의도적 미포함):**
- 블로그, 뉴스레터, Calendly, 자동 채팅 위젯, 백그라운드 비디오
- "About me" 장문 페이지, 학력 강조 섹션
- 다국어 (KO only — 타깃 100% 한국 공공기관)

## 5. The PDF one-pager (the real conversion artifact)

**파일명:** `이름_AI강의소개서_2026.pdf` (한글, 언더스코어 구분)
**규격:** A4 1페이지. PDF. B&W 출력해도 의미 전달 가능해야 함.
**선택사양:** HWP 동시 제공 (공공기관 HWP 회람 관행 대응).

**블록 순서 (이 순서 고정):**
1. 이름 · 직함 · 전문분야 한 줄
2. 액션샷 1장 + 증명사진 1장
3. 한 문단 약력 (5줄 이내)
4. 대표 강의 주제 3-4개 (제목 + 1줄 학습 결과)
5. 주요 출강 기관 (로고 또는 텍스트 리스트)
6. 강사료 정책 1줄 ("공공기관 강사료 지급기준 준수")
7. 연락처 (이메일 · 휴대폰 · 카카오톡 채널)

**폰트:** Pretendard (사이트와 동일). 본문 자간 -2~-5, 행간 1.6~1.7.
**색상:** 단색 (#1a1a1a 본문 + 1 accent). 그라데이션 금지.
**금지:** 스톡 이미지, AI 두뇌 일러스트, 영어 마케팅 카피.

## 7. Tech stack & ops

- **Framework:** Astro (정적 사이트, 빌드 빠름, 컴포넌트 재사용 가능)
- **Styling:** scoped CSS + 단일 폰트 (Pretendard)
- **Hosting:** Vercel (GitHub 연동, push → deploy)
- **Domain:** `.kr` 또는 `.co.kr` (build 단계 도메인 결정)
- **Analytics:** GA4 (PDF 다운로드 이벤트 트래킹 필수)
- **Forms:** 별도 form 없음. 카카오톡 채널 + mailto + tel: 링크만.
- **i18n:** 없음. KO 단일.

**PDF 생성:**
- 옵션 A: 디자인 툴 (Figma/Keynote)에서 한 번 만들고 `/public/소개서.pdf`로 정적 배포
- 옵션 B: Astro에서 HTML → Puppeteer로 PDF 자동생성 (오버킬, v2 고려)
- **v1 = 옵션 A 권장.** 손으로 다듬는 게 빠르고 폰트/줄간격 컨트롤이 정확함.

## 8. Legal / compliance

본 프로젝트는 **강의 서비스 제공**이며 행정사법·변호사법·세무사법 직접 적용 대상 아님.
다만 다음은 지킨다:

- "AI 정책 컨설팅" 류 단어는 신중. 공공기관 상대 "자문"은 별도 자문계약·자격 이슈가 발생할 수 있음. **본 사이트에서는 "강의/워크숍"으로만 표기.**
- 출강 사례 인용은 기관 측 명시적 동의 후 실명·로고 게재.
- 만족도 점수는 측정 출처 (자체 설문 / 기관 측정) 명시.
- "수료 예정" 자격은 **반드시 "예정" 명시.** 미수료 자격을 수료처럼 표기 시 표시광고법 이슈.

## 9. Success metrics

**Primary (v1, 첫 90일):**
- PDF 다운로드 / 월 ≥ 10건
- 카카오톡 채널 + 이메일 통합 문의 ≥ 2건/월 (목표는 1-2건이니 충분)
- 출강 전환율 (문의 → 계약) ≥ 50%

**Secondary:**
- 페이지 평균 체류 시간 60-120초 (60초 미만은 헤드라인 실패 신호)
- 모바일 트래픽 비중 ≥ 40% (주무관이 회의실/이동 중 검색)
- PDF 다운로드 / 페이지뷰 ≥ 8%

**Vanity metrics 무시:** 페이지뷰 절대값, 소셜 공유 수, 평균 스크롤 깊이.

## 10. Build plan (단계별)

**Phase 0 — Asset gather (현재 단계, 1-2주)**
- §6 콘텐츠 블루프린트 자료 모두 수집
- Hero copy 3안 작성 → Gen 컨펌
- PDF 디자인 시안 1차 (Figma)

**Phase 1 — MVP build (3-5일)**
- Astro init + Pretendard + base layout
- 8개 섹션 정적 마크업
- PDF `/public/`에 배치
- GA4 이벤트 (download / kakao / mailto / tel) 설치
- GitHub push → Vercel 자동 배포
- 도메인 연결

**Phase 2 — Polish (1-2일)**
- Lighthouse 90+ (모바일·데스크탑)
- 4G 환경 LCP < 2.5s
- 카피 최종 (AI-slop 제거 점검)
- 실제 인쇄 출력 확인 (PDF B&W)

**Phase 3 — Outreach 시작 (사이트 외 작업)**
- 콜드 아웃리치 리스트 (재단·청년센터 50곳)
- 이메일 템플릿 (사이트 + PDF 링크 첨부)
- KPI 트래킹 시작

## 13. Anti-goals (의도적으로 안 하는 것)

- 디자인 어워드 노리지 않음. **공문 친화 디자인이 목표.**
- SEO 욕심 안 부림. 콜드 아웃리치가 메인 채널.
- 강사 본인 브랜딩 강조 X. **기관이 안전하게 선택할 수 있는 강사** 인상이 핵심.
- AI 기술 자랑 X. 기술 단어는 본문에서 최소화.
