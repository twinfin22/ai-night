## 0. One-line

"AI 야학" 컨셉에 공감하는 누구든 (공공기관 주무관, 교육기관 담당자, 평생교육 코디네이터 등)이 **60초 안에** "이 강사를 결재 라인에 올릴 수 있다"고 판단하게 만드는, **outreach 이메일에 PDF가 이미 첨부된 상황**을 전제로 한 1페이지 검증 랜딩.

---

## 1. Why this exists

**기존 강사 사이트의 잘못된 가정 (1)** — "사이트가 컨버전 지점"이라고 봄.
**기존 강사 사이트의 잘못된 가정 (2)** — "방문자가 사이트에서 PDF를 처음 받는다"고 봄.

**현실** — 메인 채널은 **콜드 아웃리치 이메일**이고, PDF는 이미 메일에 첨부되어 있음. 사이트는 **"이 사람 실재하고, 우리 조직 톤에 맞고, 결재 올릴 만한가"를 60초 안에 검증하는 랜딩**.

따라서 본 사이트의 KPI는 **PDF 다운로드가 아니라 "의도 신호" — 카카오톡 채널 추가 + 메일/전화 클릭**. PDF 다운로드는 후순위(결재 첨부용 재다운로드 케이스).

## 2. Target reader & 60-second journey

### Reader ("AI 야학"에 공감할 만한 사람들)

- 비수도권 광역시·도 산하 재단 / 시·군 청년센터의 30대 주무관 (핵심)
- 고교·대학교 교직원 (교양·교직원 연수 담당)
- 평생교육원·시민대학·도서관 평생학습 코디네이터
- 공통점: 본인이 결재 권한 없음 → 팀장→과장→부장 결재 라인. "내 조직과 비슷한 사례 있음" + "단가·법규 통과"가 통과 조건.

**페르소나 검증 상태:** 출강 현장에서 담당자와 상호작용한 경험치 기반 **추정**. 인터뷰 검증 X. → §9 검증 메커니즘 참조.

### 60초 동선 (전제: outreach 이메일 클릭 후 사이트 진입, PDF 이미 받음)

```
0:00 ─ 헤더 진입 (outreach 클릭 or 검색)
0:05 ─ 헤드라인: "AI 야학" 정체성 + 한 줄 (누가/어디서/왜 가르치는지)
0:10 ─ 강의 현장 액션샷 → "실재함" 1초 검증
0:20 ─ 출강 기관 로고/리스트 → "이미 신뢰받음" 확인
0:30 ─ 대표 강의 주제 카드 → 우리 조직에 맞을 만한 주제 1개 확인
0:40 ─ 사례 (공공기관/교육기관 라벨) → 비슷한 조직 1건 매칭
0:50 ─ 강사료 1줄 (기준 준수 + 교통/숙박비) → 단가 통과 확인
0:55 ─ 카카오톡 채널 추가 OR PDF 재다운로드 (결재 첨부용)
```

이 동선 중 어느 한 칸이라도 5초 이상 걸리면 이탈. 디자인은 이 시계를 우선한다.

**v1.0 대비 변화:**
- 종착점 = "PDF 다운로드"에서 "카톡 추가 OR PDF 재다운로드"로 (의도 신호 우선)
- 헤드라인에서 "AI 야학" 정체성 명시
- 액션샷이 사례보다 먼저 (실재 검증 우선)
- 사례에 라벨 명시 (공공/교육)

## 3. Instructor positioning (the 한 줄)

- "AI 엔지니어 출신이 아니라 **눈높이를 맞추는 교육자**"
- 컨셉: **"AI 야학"** — 누구든 와서 배울 수 있는, 격식보다 이해를 우선하는 AI 교육
- 차별화 축: **비전공자·실무자 대상 AI 교육 전문성** (지방 공공기관이 primary, 교육기관·평생학습이 secondary)
- 안티-포지셔닝: "혁신적인 AI 전문가" / "차세대 AI 강사" / "열정적인 교육자"
- 톤: 공문 호환. 마케팅 영어 금지 (innovative, cutting-edge, next-generation 류 사용 금지)

**Hero headline 후보 (PRD 단계 — copy는 build 단계에서 확정):**
> "AI 야학 — 공공기관·교육기관 실무자를 위한 AI 업무자동화 강의. 누적 N회 출강, 평균 만족도 X.X"

```

│   - 강의 현장 액션샷 1장 (스튜디오 X)      │
│   - PRIMARY CTA: 카카오톡 채널 추가         │
│   - SECONDARY CTA: 강의소개서(PDF) 다운로드 │
```

## 5. The PDF one-pager (the real conversion artifact)

**파일명:** `이름_AI야학_강의소개서_2026.pdf` (한글, 언더스코어 구분)
**규격:** A4 1페이지. PDF. B&W 출력해도 의미 전달 가능해야 함.
**HWP:** v1 미제공. 첫 문의 시 케이스바이케이스 대응.

**블록 순서 (이 순서 고정):**
1. 이름 · 직함 · "AI 야학" 정체성 한 줄
2. 액션샷 1장 + 증명사진 1장
3. 한 문단 약력 (5줄 이내)
4. 대표 강의 주제 3-4개 (제목 + 1줄 학습 결과)
5. 주요 출강 기관 (로고 또는 텍스트 리스트, 공공/교육 라벨)
6. 강사료 정책 1줄 ("공공기관 강사료 지급기준 준수 · 교통/숙박비 별도")
7. 연락처 (이메일 · 카카오톡 채널 — 휴대폰 X)
8. **사이트 URL + QR 코드** (인쇄 회람 대응 필수)

**폰트:** Pretendard (사이트와 동일). 본문 자간 -2~-5, 행간 1.6~1.7.
**색상:** 단색 (#1a1a1a 본문 + 1 accent). 그라데이션 금지.
**금지:** 스톡 이미지, AI 두뇌 일러스트, 영어 마케팅 카피.

## 7. Tech stack & ops

- **Framework:** Astro (정적 사이트, 빌드 빠름, 컴포넌트 재사용)
- **Styling:** scoped CSS + Pretendard
- **Hosting:** Vercel (GitHub 연동, push → deploy)
- **Domain:** `ai-evening.kr` (메인) + `이름.kr` (보조, 리다이렉트)
- **Analytics:** GA4
  - 이벤트 추적: 카톡 채널 클릭 / 메일 클릭 / PDF 다운로드 / 외부 referrer / 체류 시간 / 진입 경로
- **Forms:** 별도 form 없음. 카카오톡 채널 + mailto만.
- **i18n:** 없음. KO 단일.

**PDF 생성:**
- 옵션 A: 디자인 툴 (Figma/Keynote)에서 한 번 만들고 `/public/소개서.pdf`로 정적 배포 — **v1 채택**
- 옵션 B: Astro에서 HTML → Puppeteer로 PDF 자동생성 (오버킬, v2 고려)

## 8. Legal / compliance

본 프로젝트는 **강의 서비스 제공**이며 행정사법·변호사법·세무사법 직접 적용 대상 아님.
다만 다음은 지킨다:

- "AI 정책 컨설팅" 류 단어는 신중. 공공기관 상대 "자문"은 별도 자문계약·자격 이슈가 발생할 수 있음. **본 사이트에서는 "강의/워크숍"으로만 표기.**
- 출강 사례 인용은 기관 측 명시적 동의 후 실명·로고 게재. (3개 사례 모두 동의 완료.)
- 만족도 점수는 측정 출처 (자체 설문 / 기관 측정) 명시.
- "수료 예정" 자격은 v1 사이트에 노출 X. 수료 후 추가. 표시광고법 리스크 회피.

## 9. Success metrics

**Primary (v1, 첫 90일) — "의도 신호" 우선:**
1. **카카오톡 채널 추가 수 ≥ 5/월** (가장 강한 의도 신호)
2. **메일 클릭 수 ≥ 3/월**
3. **Outreach 회신율** (outreach 캠페인 단위로 추적; 사이트 도메인 보유가 회신율을 올리는지 검증)
4. **PDF 다운로드 ≥ 10/월** (vanity 직전. outreach 첨부와 중복 가능성 인지)

**Secondary:**
- 페이지 평균 체류 시간 60-120초 (60초 미만 = 헤드라인 실패 신호)
- 모바일 트래픽 비중 ≥ 40%
- 카톡추가율 (카톡 클릭 / 페이지뷰) ≥ 5%

**Vanity metrics 무시:** 페이지뷰 절대값, 소셜 공유 수.

### 페르소나 검증 메커니즘 (v1.1 신규)

페르소나는 출강 담당자 상호작용 기반 **추정**이므로 빌트인 검증 필수:

- **v1 출시 후 첫 5건 문의 = 무조건 30분 인터뷰**
  - 질문: 직급, 기관 규모, 결재 경로, 사이트 도착 경로, 어느 섹션에서 결심
  - 정성 데이터로 페르소나 hypothesis 업데이트
- **GA4 로깅 추가:**
  - referrer (outreach 메일 / 검색 / 직접 입력 / 소셜)
  - scroll depth (어느 섹션에서 멈추는지 — 페르소나별 패턴 발견용)
  - 체류 시간 distribution
- **첫 30일 후 페르소나 hypothesis revision 결정 미팅 (with Claude).**
