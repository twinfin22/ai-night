# F0 /오늘마무리 스파이크 결과

## 판정

자동 파일 저장은 P0 기본 경로로 쓰지 않는다.

이유는 단순하다. Claude와 Codex 앱의 파일 접근, 폴더 선택, 승인 흐름이 수강생 환경마다 달라서 50대 초보자에게 100% 재현을 약속할 수 없다. PRD의 F0-③ 대안 경로를 채택한다.

## 이번 세션에서 확인한 것

- Claude 앱은 설치되어 있고 로그인된 새 작업 화면까지 접근 가능했다.
- Codex 앱은 설치되어 있고 실행 중이지만, Computer Use가 `com.openai.codex` 접근을 안전상 차단했다.
- 따라서 “두 앱 모두에서 `/오늘마무리` 입력 → 파일 저장”은 이 환경에서 재현 완료로 주장할 수 없다.
- 대신 Day 2와 모든 마무리 단계는 “파일 저장 시도, 실패하면 복사용 마크다운” 계약으로 바꿨다.

## 최종 제품 결정

1. Day 2에서는 `/오늘마무리`를 “마법 설치”가 아니라 “마무리 주문 저장”으로 소개한다.
2. Day 3 이후에는 자동 저장을 요구하지 않는다.
3. 성공 기준은 둘 중 하나다.
   - `내AI공부/수업일지/day-XX.md` 파일이 생김
   - 복사해서 저장할 수 있는 마크다운이 나옴
4. Day 7에서는 스킬의 정체를 “반복 양식을 저장한 문서”로 공개한다.

## No-go 항목

- “어느 앱에서나 `/오늘마무리`만 치면 파일이 자동 생성된다”는 문구 금지.
- 파일 접근 권한 설정을 초보자 필수 경로로 두는 것 금지.
- 강사가 현장에서 확인하지 않은 앱별 설치 절차를 성공 보장처럼 쓰는 것 금지.

## 남은 수동 QA

실제 수업 전에는 강사 컴퓨터에서 아래 두 경로를 한 번 더 찍어야 한다.

- Claude: 폴더 접근 권한이 있을 때 파일 생성이 되는지
- Codex: 작업 폴더 안에서 `day-XX.md` 생성이 되는지

둘 중 하나가 실패해도 제품은 출시 가능하다. fallback이 기본 경로이기 때문이다.
