# F0 매일 닫기 스파이크 결과

## 판정

자동 파일 저장은 P0 기본 경로로 쓰지 않는다.

이유는 단순하다. Claude와 Codex 앱의 파일 접근, 폴더 선택, 승인 흐름이 수강생 환경마다 달라서 초보 수강생에게 100% 재현을 약속할 수 없다. PRD의 F0-③ 대안 경로를 채택한다.

## 이번 세션에서 확인한 것

- Claude 앱은 설치되어 있고 로그인된 새 작업 화면까지 접근 가능했다.
- Codex 앱은 설치되어 있고 실행 중이지만, Computer Use가 `com.openai.codex` 접근을 안전상 차단했다.
- 따라서 “두 앱 모두에서 매일 닫기 입력 → 파일 저장”은 이 환경에서 재현 완료로 주장할 수 없다.
- 대신 모든 마무리 단계는 “오늘 만든 것과 다음에 이어 할 말을 지금 대화에 남기기”로 낮춘다.

## 최종 제품 결정

1. 별도 마무리 명령을 설치하지 않는다.
2. 모든 Day의 일반 마무리는 자동 저장을 요구하지 않는다.
3. 성공 기준은 둘 중 하나다.
   - 오늘 만든 것이 대화에 남음
   - 다음에 이어 할 말 1줄이 대화에 남음
4. Day 7에서는 스킬의 정체를 “반복 양식을 저장한 문서”로 공개한다.

## No-go 항목

- “어느 앱에서나 마무리 명령만 치면 파일이 자동 생성된다”는 문구 금지.
- 파일 접근 권한 설정을 초보자 필수 경로로 두는 것 금지.
- 강사가 현장에서 확인하지 않은 앱별 설치 절차를 성공 보장처럼 쓰는 것 금지.

## 남은 수동 QA

실제 수업 전에는 강사 컴퓨터에서 아래 두 경로를 한 번 더 찍어야 한다.

- Claude: 앱 히스토리에서 해당 주 대화가 회수되는지
- Codex: 작업 히스토리에서 해당 주 작업이 회수되는지

둘 중 하나가 실패해도 제품은 출시 가능하다. `ai-study/outputs`와 현재 대화 fallback이 있기 때문이다.
