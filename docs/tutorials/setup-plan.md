# AI 작업 환경 만들기: 리서치 노트와 구현 계획

이 문서는 `/modules/setup` 화면을 만들 때 참고할 원본 계획이다. 수강생에게는 어렵게 말하지 않는다. 목표는 "앱 설치 안내"가 아니라 **다음 수업부터 AI가 내 폴더 안에서 실제 일을 대신 시작할 수 있는 상태**를 만드는 것이다.

## 모듈 한 줄 설명

유료 계정 확인, 앱 설치, 작업 폴더 1개 만들기, 앱에 폴더 연결, 첫 정리 실습까지 끝내고 다음 수업을 이어갈 준비를 만든다.

## 수강생이 끝내야 하는 것

1. ChatGPT 또는 Claude 유료 계정 확인
2. 선택한 앱 설치와 로그인
3. `문서/Documents` 안에 작업 폴더 1개 생성
4. 앱에서 작업 폴더 열기
5. 승인/권한 요청을 읽고 허용할지 판단하는 기준 이해
6. Codex 사용자는 첫 실습으로 폴더 구조와 `AGENTS.md` 생성
7. Codex 사용자는 `AGENTS.md`에 내 규칙 1개 추가
8. 다음 시간에 붙여넣을 재개 문장 저장
9. 막혔을 때 복사해서 물어볼 문장 1개 이상 확인

## UI/UX 판단

첫 화면부터 고르는 것이 많으면 초보자는 멈춘다. 그래서 상단에는 **OS 선택 + 도구 선택**만 둔다. 선택 후에는 나에게 해당 없는 내용이 보이지 않아야 한다.

| 참고 패턴 | 가져올 점 | 이 모듈 적용 |
| --- | --- | --- |
| Codecademy식 실습 | 설명 바로 아래 행동 | 각 단계에 "내가 할 일 / AI에게 맡길 일" 배치 |
| Coursera식 완료 기준 | 끝났는지 알 수 있음 | 단계마다 눈으로 확인할 성공 기준 제공 |
| Notion 템플릿 | 필요할 때만 펼침 | 가격 비교, 폴더 이론, 고급 기능은 토글 |
| 온보딩 체크리스트 | 선택 후 다음 행동 명확 | 전역 선택기에 따라 단계 필터링 |

## 설계 원칙

- 사람이 직접 만드는 폴더는 1개만 둔다.
- 내부 폴더 구조는 AI가 만들게 한다.
- 첫날 GitHub, Vercel, 터미널, Claude Code는 제외한다.
- 기본 경로는 Codex다. Claude는 보조 경로로 둔다. 같은 수준의 자동화 성공 경험을 억지로 약속하지 않는다.
- 브라우저, Gmail, Slack, Drive 연결은 "이런 기능이 있다"까지만 보여준다.
- 토글은 "안 읽어도 끝낼 수 있는 것"에만 쓴다.
- 본문 문장은 초등학생도 따라 할 수 있게 짧게 쓴다.
- 모든 스텝은 "눈으로 확인할 것"을 포함한다.
- 설치나 권한 변경은 AI가 이유를 쉬운 말로 설명하고, 사용자가 승인한 뒤 진행한다.

## 3. 작업 폴더 1개 만들기

예상 시간: 5분

본문 문구:

> 오늘 사람이 직접 만드는 폴더는 딱 하나입니다. 나머지 정리는 AI에게 맡깁니다.

추천 폴더명:

- 기본: `ai-yahak-studio`
- 영어가 어렵다면: `AI야학-작업실`

위치:

- Mac: `문서`
- Windows: `문서/Documents`

### Mac

내가 할 일:

1. Finder를 연다.
2. 왼쪽에서 `문서`를 누른다.
3. 새 폴더를 만든다.
4. 이름을 `ai-yahak-studio`로 바꾼다.

### Windows

내가 할 일:

1. 파일 탐색기를 연다.
2. 왼쪽에서 `문서`를 누른다.
3. 새 폴더를 만든다.
4. 이름을 `ai-yahak-studio`로 바꾼다.

성공 기준:

- `문서` 안에 작업 폴더가 보인다.
- 폴더 안은 비어 있어도 괜찮다.

토글:

- 왜 바탕화면이나 다운로드 폴더를 피하는지
- 좋은 폴더명 규칙
- 폴더/파일/프로젝트/세션 차이
- OneDrive/iCloud는 왜 여기서 길게 설명하지 않는지

## 4. 앱에 작업 폴더 연결

예상 시간: 8분

### Codex 선택 시

내가 할 일:

1. Codex 앱을 연다.
2. `Open folder` 또는 `폴더 열기`를 누른다.
3. `문서/ai-yahak-studio`를 선택한다.
4. 새 작업을 시작한다.
5. 권한 모드가 읽기 전용이 아닌지 확인한다.

본문 안내:

> Codex가 파일을 만들고 고칠 수 있어야 합니다. 읽기 전용이면 다음 실습이 전부 막힙니다.

권한 안내:

- 작업실 폴더 안에 파일을 만들거나 고치는 요청은 보통 허용해도 괜찮다.
- Node, npx 같은 개발 도구 설치 요청은 이유를 읽고 허용한다.
- 관리자 비밀번호를 요구하면 바로 누르지 말고 강사나 AI에게 먼저 묻는다.
- 멈추고 읽어야 하는 경우는 3가지다.

경고 박스:

> 작업실 폴더 밖의 파일을 지우거나 바꿀 때, 내 정보를 외부로 보낼 때, 결제가 관련될 때는 바로 허용하지 말고 질문하세요.

성공 기준:

- Codex 화면에서 현재 폴더가 `ai-yahak-studio`로 보인다.
- 입력창에 메시지를 쓸 수 있다.
- 권한 모드가 파일을 만들 수 있는 상태다.

토글:

- Local / Worktree / Cloud 차이
- 승인 모드 차이
- 터미널 창이 보일 때 놀라지 않아도 되는 이유

4. 이 수업에서 만든 파일을 나중에 첨부할 수 있음을 확인한다.

본문 안내:

> Claude는 첫날 폴더 전체 연결보다 대화와 파일 첨부 중심으로 씁니다. 이 경로는 보조 옵션입니다.

## 6. 첫 실습 - Codex

예상 시간: 10분

본문 문구:

> 이제 AI에게 첫 일을 맡깁니다. 폴더 안을 직접 정리하지 말고, 아래 문장을 그대로 붙여넣으세요.

복사 버튼이 있는 프롬프트:

```text
나는 AI 야학 수강생이고 컴퓨터를 잘 몰라. 이 폴더는 내 수업 작업실이야.

시작하기 전에 먼저 확인해줘:
이 폴더가 OneDrive나 iCloud 같은 클라우드 동기화 안에 있는지 확인하고,
문제가 될 수 있으면 네가 직접 해결해줘. 해결한 내용은 쉬운 말 한 줄로만 알려줘.

그 다음, 아래 폴더 구조를 만들어줘:
- 자료함
- 결과물
- 연습
- 메모

각 폴더 안에 README.md를 하나씩 만들고, 이 폴더에 무엇을 넣으면 되는지 쉬운 말 한 줄로 적어줘.

마지막으로 AGENTS.md 파일을 만들고 이 규칙을 적어줘:
- 사용자는 터미널을 모르는 초보자다. 터미널 명령이 필요하면 사용자에게 시키지 말고
  네가 직접 실행하고, 승인이 필요하면 쉬운 말로 설명해라.
- 작업에 필요한 도구(Node, npx 등)가 없으면 왜 필요한지 쉬운 말로 설명하고, 승인받은 뒤 설치해라.
- 뭔가 막히면 (설치 실패, 권한 문제 등) 포기하지 말고 다른 방법을 찾아 시도해라.
- 사용자에게는 전문용어 없이 한두 문장으로만 설명해라.
```

AI에게 맡길 일:

- 클라우드 동기화 여부 점검
- 폴더 구조 생성
- `AGENTS.md` 생성

내가 할 일:

1. 프롬프트를 붙여넣는다.
2. 승인 요청이 뜨면 내용을 읽고 허용한다.
3. Finder/탐색기에서 파일이 생겼는지 본다.

성공 기준:

- 새 폴더들이 생겼다.
- `AGENTS.md` 파일이 생겼다.
- `AGENTS.md` 안에 위 규칙이 들어 있다.
- 현재 폴더가 `ai-yahak-studio`다.
- AI가 한 일을 쉬운 말로 설명했다.

`AGENTS.md` 설명:

> `AGENTS.md`는 AI에게 주는 규칙 노트입니다. AI는 이 폴더에서 일을 시작할 때마다 이 파일을 먼저 읽습니다. 여기에 적은 규칙은 매번 다시 말하지 않아도 계속 지켜집니다.

토글:

- 어떤 앱이 `AGENTS.md`를 읽는지
- 파일 이름을 바꾸면 안 되는 이유
- AI가 폴더를 이상하게 만들었을 때 다시 시키는 말

## 7. 내 규칙 추가하기 - Codex

복사 버튼이 있는 프롬프트:

AGENTS.md에 내 규칙을 추가해줘:

```text

```

## 8. 첫 실습 - Claude

예상 시간: 7분

본문 문구:

> Claude 사용자는 첫날 파일을 만들기보다, 앞으로 쓸 작업실 운영 규칙을 먼저 말로 정리합니다.

복사 버튼이 있는 프롬프트:

```text
나는 AI 야학 수강생이야. 앞으로 이 수업에서 자료와 결과물을 잘 정리하려고 해.
초보자 기준으로 작업실 운영 규칙을 만들어줘.

조건:
- 어려운 말은 쓰지 마.
- 내가 직접 해야 할 일과 Claude에게 맡길 일을 나눠줘.
- 다음 수업 때 이어서 쓸 수 있는 재개 문장도 만들어줘.
```

성공 기준:

- Claude가 작업실 운영 규칙을 답했다.
- 파일 첨부 버튼 위치를 안다.
- 다음 수업 재개 문장을 저장했다.
- 이 경로는 파일 생성 자동화까지 완료한 것으로 보지 않는다.

토글:

- Claude 답변을 파일로 저장하는 방법
- 나중에 Codex로 옮길 때 쓸 문장

## 9. 연결 기능은 보기만 하기

예상 시간: 3분

본문 문구:

> 오늘은 연결하지 않습니다. 이런 기능이 있다는 것만 봅니다.

보여줄 기능:

- Browser/Chrome 연결
- Gmail
- Slack
- Drive
- Plugins
- `@` 도구 호출
- `$` 스킬
- `/` 명령어

경고 박스:

> 연결 기능은 편하지만 개인정보 접근 권한이 생길 수 있습니다. 첫날에는 연결하지 않습니다.

성공 기준:

- 연결 기능의 존재만 봤다.
- 실제 계정 연결은 하지 않았다.

## 10. 마무리 체크

예상 시간: 4분

체크리스트:

- 작업 폴더가 있다.
- 앱 로그인이 끝났다.
- 앱에서 작업 폴더 또는 새 대화를 열 수 있다.
- Codex 사용자는 폴더 구조와 `AGENTS.md` 생성을 확인했다.
- Codex 사용자는 내 규칙 1개를 추가했다.
- Claude 사용자는 첫 대화와 파일 첨부 위치를 확인했다.
- 다음 시간 재개 문장을 저장했다.
- 막혔을 때 복사할 문장 1개를 찾을 수 있다.

### Codex 재개 문장

```text
내 작업 폴더는 문서/ai-yahak-studio야. 이 폴더를 기준으로 다음 실습을 이어가자.
```

### Claude 재개 문장

```text
지난 시간에 AI 야학 작업실 운영 규칙을 만들었어. 그 규칙을 기준으로 다음 실습을 이어가자.
```

저장 위치:

- Codex 사용자는 `메모/다음시간.md`에 저장한다.
- Claude 사용자는 메모 앱이나 Claude 대화 상단에 저장한다.

## 실패 복구 문구

화면 하단에 "막혔을 때 복사할 말"로 둔다.

### 앱 설치가 막힘

```text
앱 설치 중 막혔어. 지금 화면에서 무엇을 눌러야 하는지 초보자 기준으로 한 단계씩 알려줘.
```

### 폴더 연결이 막힘

```text
작업 폴더를 앱에 연결하다가 막혔어. 내가 지금 확인해야 할 것 1개만 먼저 알려줘.
```

### 승인 요청이 무서움

```text
승인 요청이 떴어. 이 요청이 안전한지 쉬운 말로 설명하고, 허용해도 되는지 판단해줘.
```

### 파일이 안 생김

```text
방금 시킨 작업 뒤에 폴더나 파일이 생기지 않았어. 원인을 확인하고 네가 다시 시도해줘.
```

## 스크린샷 계획

필수:

- Mac Finder에서 작업 폴더 생성
- Windows 탐색기에서 작업 폴더 생성
- Codex 폴더 열기 화면
- Codex 권한 모드 설정 화면
- Codex 본문 5개 요소
- Codex 승인 요청 실제 화면
- macOS Gatekeeper 경고 화면
- Windows SmartScreen 경고 화면
- Claude 첫 화면
- Claude 파일 첨부 위치

규칙:

- 실계정 캡처 사용
- 이름, 이메일, 카드, 워크스페이스명, 최근 대화 제목은 가림
- 번호 말풍선과 한 줄 캡션만 사용
- 스크린샷마다 캡처 날짜와 앱 버전을 메타데이터로 기록
- UI가 바뀌면 다시 찍어야 하는 스크린샷을 찾을 수 있어야 한다

## 구현 메모

- `/modules/setup` route로 만든다.
- 프롬프트만 따로 볼 수 있는 문서는 `docs/tutorials/setup-prompt-pack.md`로 둔다.
- 선택 상태는 `localStorage`에 저장한다.
- OS와 도구 선택 전에는 세부 단계를 숨긴다.
- 단계는 카드 남발보다 세로 체크리스트형 작업대로 만든다.
- 각 프롬프트에는 복사 버튼을 둔다.
- 복사 완료 상태는 2초만 보여준다.
- 모바일 390px에서도 프롬프트 박스가 화면 밖으로 넘치지 않아야 한다.
- 토글은 기본 닫힘이다.
- 완료 체크는 브라우저에만 저장해도 된다. 계정 기능은 만들지 않는다.

## 테스트 계획

- 왕초보가 토글을 하나도 열지 않고 끝까지 따라갈 수 있어야 한다.
- Mac에서 작업 폴더 1개 생성까지 성공해야 한다.
- Windows에서 작업 폴더 1개 생성까지 성공해야 한다.
- Codex 사용자는 폴더 연결, `AGENTS.md` 생성, 내 규칙 추가까지 성공해야 한다.
- Claude 사용자는 앱에서 첫 대화와 파일 첨부 위치를 이해해야 한다.
- Windows 실기기에서 OneDrive 리디렉션 시나리오를 실제로 1회 검증한다.
- Node/npx 없는 깨끗한 기기에서 에이전트가 스스로 설치까지 완료하는지 1회 검증한다.
- 모바일 390px에서 스크린샷, 캡션, 프롬프트 박스가 겹치지 않아야 한다.
- 외부 연결은 설명만 하고 실제 연결 과제로 만들지 않는다.

## 비판 포함

- 현재 가장 큰 리스크는 Codex 앱 UI가 자주 바뀌는 것이다. 그래서 스크린샷 메타데이터와 갱신 기준이 필요하다.
- Claude 흐름은 Codex보다 약하다. 폴더 기반 자동화가 아니라 대화/첨부 중심이라 첫 성공 경험이 덜 강하다.
- OneDrive/iCloud 문제를 AI에게 맡기는 발상은 좋지만, 초보자의 권한 승인 화면에서 멈출 가능성이 높다. 실패 복구 문구와 실제 Windows 검증이 필요하다.
- "유료 플랜 필수" 문장은 맞지만 거칠다. 결제 압박처럼 보이지 않게 무료 버전의 한계를 토글에서 설명해야 한다.
- GitHub/Vercel을 빼는 판단은 맞다. 첫날에 넣으면 설치 모듈이 아니라 개발자 온보딩이 된다.
