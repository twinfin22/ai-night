# 홈페이지 제작 순서 만들기: 프롬프트 팩

아래 문장은 그대로 복사해서 코덱스(Codex), 챗GPT(ChatGPT), 클로드(Claude)에 붙여넣는다.

## 1. office-hours 인터뷰

```text
/office-hours

나는 홈페이지를 만들고 싶어.
하지만 무엇부터 정리해야 하는지 잘 몰라.

내 상황을 먼저 인터뷰해줘.
한 번에 질문을 너무 많이 하지 말고, 1-3개씩 물어봐줘.
내 답을 듣고 홈페이지 제작 메모(pipeline_canvas)를 만들어줘.

홈페이지 제작 메모 형식:
1. 홈페이지 목적
2. 보여줄 대상
3. 방문자가 해야 할 행동
4. 꼭 들어갈 내용
5. 참고 사이트 2-3개
6. 원하는 분위기
7. 피하고 싶은 분위기
8. AI가 도와줄 일
9. 내가 직접 판단할 일
10. 첫 배포까지의 제작 순서

나는 초보자야.
어려운 말은 쉽게 풀어서 말해줘.
내가 막히면 예시를 보여주고 고르게 해줘.
```

## 2. copywriting 자료 정리

```text
/copywriting

아래 자료를 홈페이지 제작용으로 정리해줘.
나는 초보자라서 바로 다음 단계에 쓸 수 있게 정리본이 필요해.

내 홈페이지 주제:
[예: 동네 책방 소개 / 개인 포트폴리오 / 행사 모집 페이지 / 수업 신청 페이지]

참고 사이트:
1. [주소]
2. [주소]
3. [주소]

내가 가진 자료:
[소개글, 사진 설명, 가격, 일정, 연락처, 후기, 링크 등을 붙여넣기]

대상 방문자:
[누가 이 홈페이지를 보는지]

방문자가 하길 원하는 행동:
[신청하기 / 문의하기 / 예약하기 / 후원하기 / 내 작업 보기]

피하고 싶은 말투:
[예: 너무 광고 같음, 너무 AI 같음, 너무 딱딱함, 너무 과장됨]

출력 형식:
1. 홈페이지 한 줄 목표
2. 핵심 방문자
3. 방문자가 누를 버튼 문구 5개
4. 꼭 들어갈 섹션
5. 빼도 되는 내용
6. 참고 사이트에서 따라할 점
7. 참고 사이트에서 버릴 점
8. 프론트엔드 디자인(frontend-design)에 넘길 제작 요청문
```

## 3. frontend-design 제작

```text
/frontend-design

아래 제작 요청서를 바탕으로 홈페이지를 만들어줘.
목표는 예쁜 샘플이 아니라 실제로 배포할 수 있는 첫 버전이야.

홈페이지 제목:
[제목]

홈페이지 목표:
[한 줄 목표]

대상 방문자:
[방문자]

버튼/신청 행동(CTA, 방문자가 누를 것):
[방문자가 누를 버튼 또는 해야 할 행동]

꼭 들어갈 섹션:
[섹션 목록]

참고 사이트:
[주소 2-3개와 참고할 점]

피하고 싶은 것:
[AI 티 나는 문구 / 과장된 문구 / 너무 화려한 디자인 / 모바일 깨짐 등]

요청:
1. 첫 화면에서 무엇을 하는 곳인지 바로 보이게 해줘.
2. 모바일에서도 읽기 쉽게 만들어줘.
3. 버튼은 찾기 쉽게 만들어줘.
4. 문구는 한국어로 쉽고 자연스럽게 써줘.
5. 필요한 파일을 수정하고, 실행 방법을 알려줘.
6. 끝나면 디자인 리뷰(design-review)에 넘길 혹평 요청문도 만들어줘.
```

## 4. design-review 혹평

```text
/design-review

이 홈페이지를 좋게 포장하지 말고 리뷰해줘.
초보자가 실제 배포 전에 고쳐야 할 문제를 찾아줘.

검토할 것:
1. 첫 화면에서 무엇을 하는 곳인지 3초 안에 보이는가
2. 방문자가 눌러야 할 버튼이 분명한가
3. 문구가 AI처럼 뻔하거나 과장되어 보이는가
4. 모바일에서 글자, 버튼, 섹션이 깨지지 않는가
5. 신뢰를 주는 정보가 충분한가
6. 필요 없는 장식이 많은가
7. 단체방에 공유하기 전에 꼭 고쳐야 할 치명적인 문제가 있는가

출력 형식:
1. 가장 큰 문제 3개
2. 바로 고칠 문구
3. 바로 고칠 화면
4. 모바일 확인 포인트
5. 수정 우선순위
6. 수정용 프롬프트

칭찬은 짧게.
고칠 점을 구체적으로 말해줘.
```

## 5. Playwright 확인

```text
/playwright

이 홈페이지를 실제 브라우저에서 확인해줘.

필수 확인:
1. 화면 없이 자동으로 여는 브라우저(headless, 자동 확인 방식)로 데스크톱 화면 확인
2. 화면 없이 자동으로 여는 브라우저(headless, 자동 확인 방식)로 모바일 화면 확인
3. 첫 화면에서 정체가 바로 보이는지
4. 버튼이 눌리는지
5. 내부 링크가 열리는지
6. 글자나 버튼이 겹치거나 잘리지 않는지
7. 배포 주소가 있으면 정상 열림(200, 문제없이 열림) 상태인지

스크린샷은 필요할 때만 잠깐 확인하고, 파일로 저장하지 않아도 돼.
문제가 있으면 통과라고 하지 마.
고칠 파일과 다음 행동 하나만 알려줘.
```

## 6. Vercel 배포

```text
@vercel

이 홈페이지를 Vercel(버셀)로 배포해줘.
나는 초보자야.

먼저 필요한 연결 상태를 확인해줘.
계정 연결이 필요하면 어디를 눌러야 하는지 한 단계씩 알려줘.
연결이 끝나면 미리보기 주소(preview URL, 먼저 확인해보는 주소)를 만들어줘.
실패하면 원인을 쉬운 말로 설명하고 다음 행동 하나만 알려줘.
```

## 7. GitHub(깃허브) 설정 보너스

```text
나는 GitHub(깃허브)를 처음 써.
내 홈페이지 프로젝트를 GitHub(깃허브)에 올리고 Vercel(버셀)과 연결해서,
앞으로 수정하면 자동 배포되게 만들고 싶어.

한 번에 한 단계씩만 안내해줘.
내가 "완료"라고 답하면 다음 단계로 넘어가줘.
어려운 용어는 쓰지 말고, 꼭 필요하면 바로 뜻을 설명해줘.

최종 목표:
GitHub(깃허브) 계정 생성 -> 새 저장소 만들기 -> 프로젝트 올리기 -> Vercel(버셀)에서 가져오기(Import) -> 자동 배포 확인.
```

## 중간에 멈출 때만 쓰는 20분 재개

```text
지금까지 한 일을 요약해줘.
그리고 다음 세션 시작 때 그대로 붙여넣을 프롬프트를 만들어줘.

포함할 것:
1. 현재 목표
2. 완성된 것
3. 아직 안 된 것
4. 막힌 점
5. 다음 20분에 할 일 하나
6. 다음에 쓸 추천 스킬
7. 현재 배포 주소가 있으면 포함
```
