# 홈페이지 제작 순서 만들기: 리서치 노트

이 문서는 `/modules/website` 화면을 만들 때 참고할 원본 노트다. 수강생에게는 어렵게 말하지 않는다. 목표는 "홈페이지 만들기"가 아니라 "내가 다음에도 반복할 수 있는 제작 순서 만들기"다.

## 모듈 한 줄 설명

AI와 대화해서 내 상황에 맞는 홈페이지 제작 순서를 만들고, 실제 Vercel(버셀) 배포 주소를 커뮤니티 단체방에 공유한다.

## 수강생이 끝내야 하는 것

1. 홈페이지 제작 메모(pipeline_canvas)
2. 참고 사이트 2-3개와 내 자료 정리
3. 홈페이지 초안
4. 혹평 받은 뒤 수정한 결과
5. Playwright(플레이라이트) 확인 결과
6. Vercel(버셀) 배포 주소
7. 커뮤니티 단체방에 Vercel(버셀) 배포 주소 공유

## UI/UX 판단

기존 구성은 학습 모듈보다 긴 설명 페이지에 가까웠다. 그래서 구조를 `준비물 -> 단계별 토글 -> 공유`로 바꾼다.

훔칠 패턴이 아니라 따라할 패턴:

| 참고 패턴 | 가져올 점 | 이 모듈 적용 |
| --- | --- | --- |
| Coursera식 강의 | 단계, 예상 시간, 완료 기준 | 각 단계에 시간과 결과물 표시 |
| Codecademy식 실습 | 설명과 행동을 가까이 배치 | 단계 안에 "내가 할 일 / AI에게 맡길 일" 배치 |
| Notion 템플릿 | 펼치고 닫으며 따라가는 작업대 | 각 단계와 프롬프트를 토글로 숨김 |
| Khan Academy식 연습 | 짧은 설명 뒤 바로 실습 | 긴 이론보다 바로 복붙 프롬프트 제공 |
| Duolingo식 진행 | 작은 단위와 즉시 피드백 | 6단계로 만들고 마지막에 단체방 공유 |

## 설계 판단

### 긴 설명보다 작업대

첫 화면부터 "이 모듈에서 무엇을 끝내는지"와 "얼마나 걸리는지"를 보여준다. 프롬프트 전체를 한꺼번에 보여주면 초보자는 압박을 느끼므로, 각 단계 안에 접어 둔다.

### 추천 스킬은 단계 안에 둔다

"꼭 쓰면 좋은 기능"을 별도 섹션으로 빼면 학습자가 또 하나의 과제로 느낀다. 그래서 Lighthouse(라이트하우스), axe(액스), GitHub(깃허브), Vercel logs(버셀 로그)는 관련 단계 안의 "같이 쓰면 좋은 기능"으로 넣는다.

### 사람과 AI 역할 분리

AI가 할 일:

- 질문하기
- 자료 정리하기
- 구조 제안하기
- 초안 만들기
- 좋게 포장하지 않고 문제 찾기
- Playwright(플레이라이트)로 실제 브라우저 확인하기
- 배포 실패 원인 쉽게 설명하기

사람이 판단할 일:

- 누구에게 보여줄 홈페이지인지 결정
- 어떤 말투가 싫은지 말하기
- 최종 문구가 내 말처럼 보이는지 확인
- 배포 주소가 실제로 열리는지 확인
- 모바일/데스크톱 화면에서 깨진 부분을 직접 확인
- 수정 전/후 중 어느 쪽이 나은지 고르기

### Playwright(플레이라이트) 확인 필수

눈으로 한 번 보는 것과 실제 브라우저에서 눌러보는 것은 다르다. Playwright(플레이라이트)는 화면 없이 자동으로 여는 방식(headless, 자동 확인 방식)으로 써도 된다. 스크린샷은 저장하지 않아도 된다.

### 배포 필수

내 컴퓨터에서만 보이는 화면은 공유할 수 없다. 실제 링크가 열려야 한다. 기본은 Vercel(버셀) 미리보기 주소(preview URL, 먼저 확인해보는 주소)다. GitHub(깃허브) 자동 배포는 보너스다.

## 단계별 추천 스킬

| 단계 | 추천 스킬 | 같이 쓰면 좋은 기능 |
| --- | --- | --- |
| 내 상황 말하기 | office-hours(오피스아워) | spec(스펙) |
| 자료 모으기 | copywriting(카피라이팅) | GitHub(깃허브) |
| 첫 화면 만들기 | frontend-design(프론트엔드 디자인) | ui-mockup(유아이 목업), ts-react(타입스크립트 리액트) |
| 혹평받고 고치기 | design-review(디자인 리뷰) | web-design-guidelines(웹 디자인 가이드), copywriting(카피라이팅) |
| 브라우저 확인 | Playwright(플레이라이트) | Lighthouse(라이트하우스), axe(액스) |
| 배포하기 | Vercel(버셀) | Vercel logs(버셀 로그), GitHub(깃허브) |
| 공유 | 없음 | 없음 |

## 화면에 꼭 보여줄 문장

- "목표는 홈페이지 하나가 아니라, 다음에도 반복할 제작 순서를 만드는 것입니다."
- "빈칸을 혼자 채우지 마세요. AI에게 먼저 인터뷰를 시키세요."
- "공유할 것은 이미지가 아니라 실제로 열리는 Vercel(버셀) 배포 주소입니다."
- "첫 결과물은 거의 항상 부족합니다. 좋게 포장하지 않는 리뷰를 받고 고친 뒤 공유하세요."
- "배포 전에 Playwright(플레이라이트)로 데스크톱/모바일과 버튼 클릭을 확인하세요."
- "중간에 멈출 때만 20분 재개 프롬프트를 만들고 끝내세요."
