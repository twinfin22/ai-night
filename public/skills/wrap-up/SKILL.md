# /wrap-up

## 매일 모드

사용자가 `/wrap-up`이라고 말하면 오늘 수업을 수업일지로 정리한다.

가능하면 `ai-study/journal/day-XX.md` 파일로 저장한다. `XX`는 두 자리 Day 번호다. 예: `day-01.md`

Day 번호가 없으면 먼저 물어본다.

파일을 직접 만들 수 없으면 실패하지 말고, 사용자가 복사할 수 있게 아래 마크다운만 보여준다.

```markdown
# Day XX 수업일지 (YYYY-MM-DD)

## 오늘 만든 것
-

## 잘 통한 부탁 방식
-

## 안 통한 부탁 방식
-

## AI가 나에 대해 새로 알게 된 사실 1개
-

## 내일 이어할 말
-
```

## 주간 모드

사용자가 `/wrap-up 주간`이라고 말하면 이번 주 수업일지를 읽고 `ai-study/AGENTS.md`에 넣을 후보를 제안한다.

진행 순서:

1. `ai-study/journal/`에서 이번 주 Day 파일들을 읽는다.
2. "AI가 나에 대해 새로 알게 된 사실 1개"와 "잘 통한 부탁 방식"을 모은다.
3. `ai-study/AGENTS.md`에 추가하거나 고칠 후보를 짧게 보여준다.
4. 사용자가 승인한 항목만 반영한다.

## 규칙

- AGENTS.md에는 제가 승인한 것만 반영한다.
- 추측한 내용은 AGENTS.md에 넣지 않는다.
- AGENTS.md는 짧게 유지한다.
- 어려운 말은 쓰지 않는다.
- 파일 저장 실패를 길게 설명하지 않는다.
- 같은 Day를 다시 실행하면 같은 파일을 덮어써도 된다고 안내한다.
